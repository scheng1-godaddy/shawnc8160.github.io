This app was actually my wife’s idea. She has an obsession with planning and planners. FamilyPlanner is essentially a group calendar. You can create custom categories (like work, birthdays, etc) and assign colors to these categories.

This app was built using node/express for the backend RESTful API. Postgres is used for storage and React is used for the front end. Depending on how big this gets I might end up using redux.

Obviously, it’s still a work in progress as there’s no front end for user creation (but the backend is there!). Some of the next steps is to allow for reoccurring events (schedule for weekly, monthly, etc). User input validation is also high on the list.

ComicStack

ComicStack was one of my group projects while at General Assembly. My partner and I built this in 4 days and we had a ton of fun building this. We both loves comics and thought: wouldn’t it be cool to create a site to keep track of our comic collection — and the result was ComicStack. The concept is simple, you create an account, search for comic books and add it to your virtual collection. You can also keep a listing of your favorite characters.

ComicStack uses a RESTful API in the backend that was built with Ruby on Rails. Using various gems for JWT authentication and external API calls. The external API we're using is provided by ComicVine (one of my favorite API’s so far). We’re using Postgres as the database and the frontend uses React. For CSS we ended up using the bulma framework.

Something for the future might be the ability to browse other collections. Maybe finding suggestions for new comics based on the users current collection. Definitely looking forward to working on this more.

PlaylistR
AngularJS
Express
Node
MongoDB
Open Site
See Code

PlaylistR’s concept is simple. You create custom playlists of your favorite music. You can find your music by searching for tracks or artist. You can view playlists created by other users. PlaylistR was the result of another group project (with 2 other contributors) during my tenure at General Assembly.

The app was built using the MEAN stack (Mongo, Express, Angular and Node). The backend uses RESTful routes (Node/Express) and stores its data in MongoDB. It also relies heavily on the lastFM API. When a user searches for track or artist we make an API call to lastFM and pull the track information from their catalog. When the user actually adds or remove a track we make a call to our REST API and store that in mongo. The frontend uses AngularJS. One of my favorite features (thanks to Angular) was the ability to filter playlists by tags. You could filter by artist, track name or other tags like genre and era (we’re pulling the tags from lastFM and storing that in the collection!).

Something for the future might be a way to actually stream the music from the app itself.

What’s for dinner — in my house, that’s a question that gets asked nightly. The goal for “What’s for Dinner?” Is to allow people to share what they’re having for dinner and browse what others are having for dinner. Users can create a profile, post what they’re having for dinner and browse other posts and user profiles. Users can like other posts.

The app was created using mongo, express and node. It uses ejs templates to render pages. It uses bootstrap for css framework. The current version was the result of 4 days of work. There’s multiple changes I’d like to make, some of which include: allowing users to actually upload images (they’re asked to enter image URL) and user input validation. More to come soon!

Term tracker was my final project for a mobile development course at WGU. It lets students add courses to their current term (or semester).
